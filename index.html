01 · Descripción general

Una tubería completa, de la API al dashboard.

Diseño e implementación de un pipeline completamente automatizado que recopila comentarios públicos de las cuatro principales redes sociales, los limpia, los enriquece con análisis de sentimiento y los entrega listos para consultar en una base de datos PostgreSQL en la nube.

Los datos procesados alimentan dashboards de inteligencia de negocio para el equipo de analítica y sirven como fuente de entrenamiento e inferencia para modelos de IA/ML. Una interfaz React + FastAPI permite a los usuarios autorizados configurar la recopilación, monitorear ejecuciones y explorar resultados — todo alojado en infraestructura AWS.

02 · Alcance y límites

Qué entra, qué se queda fuera.

Dentro del alcance

  • Recopilación de comentarios vía APIs oficiales de YouTube, Meta y TikTok
  • Almacenamiento bruto en AWS S3 — JSON con marca de tiempo
  • Limpieza Python: normalización, deduplicación, eliminación de nulos y HTML
  • Análisis de sentimiento con HuggingFace Transformers
  • Carga en PostgreSQL sobre AWS RDS
  • Interfaz React + FastAPI para configurar y ejecutar pipelines
  • Orquestación con Prefect — programación, reintentos, alertas
  • Dashboards Metabase conectados a RDS
  • Despliegue en AWS (EC2, S3, RDS, Secrets Manager)
  • Código versionado en GitHub con documentación

Fuera del alcance

  • Contenido privado: DMs, Stories o cualquier dato no público
  • Streaming en tiempo real — el pipeline se ejecuta en intervalos
  • Datos de publicidad pagada o dashboards nativos de plataforma
  • Sentimiento multilingüe en v1 — solo inglés inicialmente
  • Aplicación móvil — solo interfaz web

04 · Stack tecnológico

Herramientas elegidas con criterio.

A Ingesta de datos

  • YouTube Data API v3Gratuita con cuota generosa. Endpoint de hilos de comentarios.
  • Meta Graph APIInstagram y Facebook. Requiere revisión de la app.
  • TikTok Research APIRequiere aprobación empresarial o académica.
  • Python · requestsAuth, paginación y límites de tasa.

B Procesamiento

  • Python 3.11+Lógica del pipeline y del backend.
  • pandas / polarsLimpieza, transformación y normalización a escala.
  • HuggingFace Transformerscardiffnlp/twitter-roberta-base-sentiment
  • SQLAlchemyORM Python para escribir filas en PostgreSQL.

C API Backend

  • FastAPIEndpoints que consume el frontend.
  • PydanticValidación de modelos de solicitud y respuesta.
  • UvicornServidor ASGI sobre EC2.

D Frontend / UI

  • React 18Selector de plataforma, historial de ejecuciones, tabla de resultados.
  • TailwindCSSEstilo rápido, limpio y responsivo.
  • AxiosCliente HTTP hacia el backend FastAPI.

E Infraestructura y orquestación

  • AWS EC2Servidor Linux siempre activo.
  • AWS S3Data lake bruto permanente.
  • AWS RDSPostgreSQL gestionado.
  • AWS Secrets ManagerAPI keys y credenciales seguras.
  • PrefectProgramación, reintentos y alertas.

05 · Fases y cronograma

Tres semanas, seis fases, una entrega.

  1. Semana 1 Fase 01

    Configuración de APIs y scripts de ingesta

    • Solicitar acceso a Meta Graph API y TikTok Research API
    • Scripts Python con paginación y gestión de rate limits
    • JSON en bruto en S3 con estructura por marca de tiempo
    • Tests unitarios por módulo de ingesta
  2. Semana 1 Fase 02

    Pipeline de limpieza y sentimiento

    • Deduplicación, eliminación de nulos, HTML stripping
    • Modelo HuggingFace para puntuación pos/neg/neutro
    • Carga de filas enriquecidas en PostgreSQL
    • Tests unitarios y de integración
  3. Semana 2 Fase 03

    Configuración de infraestructura AWS

    • Provisionar EC2, RDS y buckets S3
    • Secrets Manager para API keys y credenciales de BD
    • Roles IAM, security groups y redes VPC
    • Despliegue del pipeline en EC2 y verificación E2E
  4. Semana 2 Fase 04

    Orquestación con Prefect

    • Flujos y tareas con dependencias correctas
    • Ejecuciones programadas (cada 6 h por plataforma)
    • Alertas Slack y email ante fallos o anomalías
    • Pruebas de reintentos y recuperación ante fallos
  5. Semana 3 Fase 05

    Interfaz web React + FastAPI

    • Endpoints para disparar scrapes, consultar resultados y estado
    • Frontend: selector, input de cuenta, historial, tabla de resultados
    • Conexión React ↔ FastAPI vía Axios sobre EC2
    • Autenticación básica para proteger la interfaz
  6. Semana 3 Fase 06

    Dashboards, pruebas y entrega

    • Metabase conectado a RDS — tendencias, volumen, top comentarios
    • Pruebas extremo a extremo en las 4 plataformas
    • Documentación: setup, arquitectura, runbook
    • Walkthrough con el equipo de analítica
